- **Primary pain:** Vendor lock-in (cloud), GPU availability, OAuth complexity (6 weeks of pain), centralized LLM governance
- **Buyer profile:** Light technical teams who want a partner to handle management — not just software
- **Decision motion:** Highly motivated buyers ("tomorrow if we find the right product") — speed of POC matters
- **Competitive reality:** Databricks is already approved internally at Fortescue. Every deal needs a crisp "Why TrueFoundry vs. Databricks" narrative.
- **Pricing signal:** $150K-$350K deal size. Pricing scalar is confusing customers. Simpler = faster.

### Product Fluency You Now Have
- **AI Gateway:** Lightweight routing layer → connects apps to LLMs/Tools/Agents → governance, monitoring, control

- **MCP Gateway:** Key differentiator — handles OAuth offloading, token exchange. Exactly what Fortescue needs.
- **Control plane** = logs, monitoring, orchestration | **Compute plane** = where GPU jobs run
- **No TrueFoundry SDK required** = vendor lock-in prevention — a massive, undersold differentiator
- **Multi-cloud:** Works across AWS, GCP, Azure + on-prem/physical servers
